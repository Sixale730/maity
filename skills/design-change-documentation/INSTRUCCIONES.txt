# INSTRUCCIONES: Design Change Documentation

## Propósito
Mantener un historial transparente de por qué se hicieron los cambios de diseño.

## Cómo usar
1.  **Registro**: Cada vez que se termine un cambio, el agente actualizará el log.
2.  **Consulta**: Revisa `resources/output/DESIGN_LOG.md` para ver el historial.

## Estructura de Carpetas
- `SKILL.md`: Estándares de documentación.
- `resources/output`: El archivo de log definitivo.
